MenuHolograms — меню в мире

Плагин создаёт “менюшки” прямо в мире Minecraft: текст, предметы, блоки и “картинки” (через resourcepack). Взаимодействие выполняется через кликабельные хитбоксы (Interaction): поддерживаются hover, клики ЛКМ/ПКМ, анимации и управление элементами по id.

Hover = луч по хитбоксам

Наведение считается по Interaction-энтити, а не по “визуалу”. Если нужно — расширяй hitbox/extra_hitboxes.

Настройки меню — в MenuDefinition

Важные настройки можно задавать в menus/*.yml и они перекрывают config.yml.

Анимации — отдельные файлы

Папка animations/, форматы: TEXT/BLOCK/IMAGE, режимы: single/repeat/lastframe.

Element IDs

Каждому элементу можно дать id и управлять им через action element:.

Требования

  • Minecraft: версии, где доступны Display и Interaction сущности (как правило, 1.19.4+).
  • Сервер: Paper/Spigot совместимый сервер (рекомендуется Paper).
  • Resourcepack: нужен только если используешь элементы IMAGE.
Совет: если что-то “не кликается” или “hover не ловится”, почти всегда причина в размерах/позиции хитбокса или в расстоянии.

Установка

  1. Скопируй Menuholograms.jar в папку plugins/.
  2. Запусти сервер: плагин создаст папки menus/, animations/ и т.д.
  3. Создай меню-файл (например plugins/Menuholograms/menus/main.yml).
  4. Перезапусти или сделай reload командой плагина (если есть).

Структура папок

plugins/Menuholograms/
  config.yml
  menus/
    main.yml
    shop.yml
  animations/
    logo_anim.yml
    welcome_text.yml
  images/
    (опционально — исходники/ассеты)
  resourcepack/
    (выход/сборка — зависит от ResourcePackService)
Важно: все файлы меню и анимаций — обычные .yml. Имя меню/анимации берётся из имени файла.

config.yml — обзор

config.yml хранит глобальные значения по умолчанию: префикс, поведение динамического anchor (teleport), авто-закрытие, ресурс-пак. Любые параметры, заданные в menus/*.yml, перекрывают config.yml (через MenuDefinition). 

# пример (структура может отличаться)
prefix: "&7[MH]&f"

teleport:
  player_distance: 2.0
  looping: false
  loop_tick: 1
  chase_player: false
  tp_tick_duration: 1

close_distance: 32.0
close_time: "-1"

resourcepack:
  enabled: false

config.yml — параметры (каждый параметр + как работает)

Параметр Тип Значение по умолчанию Что делает (как работает)
prefix string §7[MH] + пробел Префикс для сообщений плагина. Применяется через color() (поддержка &, HEX, gradient).
teleport.player_distance double 2.0 Дистанция от игрока до динамического меню по направлению yaw (по горизонтали). Если в меню нет anchor, меню появляется “перед игроком”.
teleport.looping boolean false Если true — меню будет обновлять anchor по таймеру и “двигаться” вместе с игроком/камерой (в зависимости от chase_player).
teleport.loop_tick int 1 Как часто пересчитывать anchor (в тиках). 1 = каждый тик, 20 = раз в секунду.
teleport.chase_player boolean false Если true — меню будет следовать за поворотом камеры (yaw). Если false — направление фиксируется при открытии (меню остаётся в одной “стороне” от игрока).
teleport.tp_tick_duration int 1 Интерполяция телепорта Display-энтити (сколько тиков “плавно доезжает” до новой позиции). 0 = мгновенно.
close_distance double 32.0 Дистанция авто-закрытия, если close_time = -1. Проверяется как distanceSquared до anchor.
close_time string -1 Таймер авто-закрытия после последнего взаимодействия. Формат: "-1" (выкл), "10s", "500ms", "1m", "1h" или просто число (секунды).
resourcepack.enabled boolean false Если true — запускается сервис сборки/раздачи ресурс-пака (ResourcePackService). Нужен для IMAGE и кастомных моделей.
Перекрытие: многие параметры можно переопределить в конкретном меню (см. teleport/close в menu.yml).

Цвета / градиенты

Плагин использует метод Menuholograms.color() для форматирования строк:

  • &a и стандартные коды цветов Minecraft.
  • HEX: #RRGGBB или &#RRGGBB.
  • Градиент: <gradient:#FF0000:#00FF00>Текст</gradient>.
Удаление цветов: для расчёта авто-хитбокса у TEXT используется “инвертированный” метод (часто называют uncolor/strip) — он убирает все секции §x/§a и т.п., чтобы длина текста считалась правильно. 
message: "<gradient:#FF0000:#00FF00>Hello</gradient> &7world #33CCFF!"

menus/*.yml — формат файла меню

Меню — это yml файл в папке plugins/Menuholograms/menus. Имя меню = имя файла без .yml. 

# plugins/Menuholograms/menus/main.yml

# (опционально) фиксированная точка спавна меню
anchor:
  world: world
  x: 0
  y: 100
  z: 0

# (опционально) переопределение teleport из config.yml
teleport:
  player_distance: 2.0
  looping: true
  loop_tick: 1
  chase_player: true
  tp_tick_duration: 1

# (опционально) переопределение авто-закрытия
close_distance: 32.0
close_time: "10s"

# элементы
elements:
  - id: title
    type: TEXT
    x: 0
    y: 0.7
    z: 0
    text: "&bMenuHolograms"

  - id: play
    type: TEXT
    x: 0
    y: 0.4
    z: 0
    text: "&aИграть"
    left:
      - "sound: UI_BUTTON_CLICK 1 1"
      - "command: warp spawn"

anchor

anchor определяет, где меню находится в мире.

Если anchor задан
Меню всегда спавнится на указанной точке мира. Не “следует” за игроком.
Если anchor НЕ задан
Меню спавнится относительно игрока:
  • по горизонтали — на player_distance по yaw
  • по высоте — Y игрока + 1
  • может обновляться по таймеру (looping)
anchor:
  world: world
  x: 10.5
  y: 64
  z: -20.5

teleport (настройки динамического anchor)

Эти параметры можно задавать в menu.yml. Если параметр не указан — берётся значение из config.yml.

Параметр Тип Где действует Как работает
teleport.player_distance double только если anchor отсутствует Смещение меню от игрока по направлению yaw (только XZ). При открытии вычисляется “вперёд”.
teleport.looping boolean только если anchor отсутствует Если true — anchor пересчитывается каждые loop_tick тиков.
teleport.loop_tick int только при looping=true Частота пересчёта anchor. Чем больше — тем “реже” меню обновляет положение.
teleport.chase_player boolean только при looping=true true — направление (XZ) пересчитывается по yaw камеры. false — направление фиксируется в момент открытия меню.
teleport.tp_tick_duration int всегда (для Display) Плавность перемещения Display при teleport() (interpolation duration в тиках). 
teleport:
  player_distance: 2.0
  looping: true
  loop_tick: 2
  chase_player: true
  tp_tick_duration: 1

Авто-закрытие (close_time / close_distance)

Логика закрытия:

  • Если close_time задан и не равен -1 — меню закроется через указанное время после последнего взаимодействия.
  • Иначе — меню закрывается по дистанции close_distance от anchor.
# закрыть через 15 секунд после последнего клика/hover
close_time: "15s"

# закрывать по расстоянию, таймер выключен
close_time: "-1"
close_distance: 24.0

elements — элементы меню

Элементы — это список объектов. Каждый элемент может быть кликабельным и иметь hover. Рекомендуется всегда задавать id, чтобы потом управлять элементом через actions.

Типы элементов

type Сущность content поле Комментарий
TEXT TextDisplay text Текст поддерживает цвета/градиенты через Menuholograms.color().
ITEM ItemDisplay material Отображает обычный предмет Minecraft.
BLOCK BlockDisplay material Отображает блок (BlockData создаётся из Material).
IMAGE ItemDisplay (PAPER) image “Картинка” — это PAPER с item model: menuholograms:<imageId>.
Требует ресурс-пак с нужными моделями.

Параметры элемента (полный разбор)

Параметр Тип Обязателен Как работает
id string нет, но рекомендуется Идентификатор элемента. Используется в action element: для управления другими элементами. Если не задан — плагин присвоит авто-id (например e0, e1), но на него неудобно ссылаться.
type enum да Тип элемента: TEXT / ITEM / BLOCK / IMAGE.
x, y, z double да Позиция относительно anchor. Дополнительно: меню разворачивается лицом к игроку, поэтому x/z “поворотятся” вместе с меню.
scale double нет Масштаб Display. Влияет на авто-хитбокс (если hitbox не задан вручную).
rotation number или object нет Локальный поворот элемента (не путать с поворотом меню к игроку!). Применяется через Transformation (Quaternion) к Display.
text string только для TEXT Текст (поддержка цветов/градиентов). Можно многострочно через \n.
material string только ITEM/BLOCK Material по имени (например DIAMOND, GOLD_BLOCK).
image string только IMAGE item model id. В ресурс-паке это соответствует menuholograms:<imageId>.
animation string нет Имя файла анимации из папки animations/ без расширения.
animation_autostart boolean нет Запускать анимацию при открытии меню. Если не задан — считается true.
hitbox object нет Основной Interaction-хитбокс (ширина/высота/смещение). Если не указан — авто.
extra_hitboxes list нет Дополнительные Interaction-хитбоксы с id и локальными смещениями.
left list<string> нет Actions при ЛКМ по элементу или по его хитбоксам.
right list<string> нет Actions при ПКМ по элементу или по его хитбоксам.
hover object нет Настройки визуала/хитбоксов/анимаций при наведении.
onHover list<string> нет Actions при входе hover (курсор попал в хитбокс).
onUnhover list<string> нет Actions при выходе hover.

rotation — как работает поворот

Есть 2 разных поворота:

  • Поворот меню к игроку — каждый тик вычисляется yaw до игрока и применяется к entity rotation (yaw).
  • rotation элемента — локальный поворот Display через Transformation (yaw/pitch), добавляется к базовому трансформу.
Практически: rotation полезен, чтобы “наклонить” картинку или повернуть предмет, не ломая общий поворот меню. 
# коротко: только yaw
rotation: 45

# полно:
rotation:
  yaw: 20
  pitch: -10

hitbox — основной хитбокс

Основной хитбокс — это сущность Interaction: её “ширина/высота” определяет область, куда должен попасть rayTrace для hover и клика. 

hitbox:
  width: 1.2
  height: 0.6
  yOffset: 0.0
Если hitbox не задан: плагин пытается подобрать размеры автоматически:
  • TEXT — по длине строк (без цветов) и количеству строк
  • ITEM — ~0.75 * scale
  • BLOCK/IMAGE — ~1.0 * scale

extra_hitboxes — дополнительные хитбоксы с id

Дополнительные хитбоксы нужны, когда:

  • у элемента сложная форма и один hitbox не подходит
  • надо расширить область hover/клика
  • надо делать “несколько зон” клика на одном объекте (но логика клика всё равно общая: left/right этого элемента)
extra_hitboxes:
  - id: leftZone
    x: -0.5
    y: 0.0
    z: 0.0
    width: 0.7
    height: 0.7

  - id: rightZone
    x: 0.5
    y: 0.0
    z: 0.0
    width: 0.7
    height: 0.7

Изменение/удаление extra hitboxes на hover

В hover.extra_hitboxes можно: удалить существующий по id, изменить параметры, добавить новый. 

hover:
  extra_hitboxes:
    - id: leftZone
      remove: true     # убрать при наведении

    - id: newZone      # добавить при наведении
      x: 0
      y: 0
      z: 0.3
      width: 1.4
      height: 0.25
Важно: hover считается по хитбоксам. Если “hover не ловится” — увеличь width/height или добавь extra_hitboxes ближе к визуалу.

hover — настройки при наведении

Hover срабатывает, когда луч игрока попадает в любой hitbox элемента (primary или extra). При входе hover: применяется визуал, меняются хитбоксы и выполняются actions. 

hover:
  glow: true
  scale: 1.1

  rotation:
    yaw: 10
    pitch: 0

  text: "&eНавёлся!"
  material: DIAMOND_BLOCK
  image: "logo_hover"

  hitbox:
    width: 2.0
    height: 0.6
    yOffset: 0.0

  animation_action: pause
Параметр hover Тип Как работает
hover.glow boolean Включает glowing у Display при наведении.
hover.scale double Домножает базовый scale элемента при наведении.
hover.rotation number или object Добавляется к базовой rotation через Transformation.
hover.text string Для TEXT: заменяет текст при наведении.
hover.material Material Для ITEM/BLOCK: меняет предмет/блок при наведении.
hover.image string Для IMAGE: меняет imageId (item model) при наведении.
hover.hitbox object Изменяет размеры primary hitbox при наведении.
hover.extra_hitboxes list Изменяет/добавляет/удаляет extra_hitboxes при наведении.
hover.animation_action string Управляет анимацией элемента при наведении (см. раздел “Анимации → Управление”).

Actions — как работают

Actions — это строки в списках left, right, onHover, onUnhover.

Замена плейсхолдеров: поддерживается %player% → ник игрока. 
left:
  - "sound: UI_BUTTON_CLICK 1 1"
  - "message: &aПривет, %player%!"
  - "command: warp spawn"

Actions — полный список

Action Формат Что делает
sound sound: <SOUND> [vol] [pitch] Проигрывает звук игроку. Пример: sound: UI_BUTTON_CLICK 1 1.
message message: <text> Отправляет сообщение игроку (через prefix). Текст проходит через color().
command command: <cmd> Выполняет команду от имени игрока.
console console: <cmd> Выполняет команду от консоли.
open open: <menu> Открывает другое меню.
close close Закрывает текущее меню.
element element: <id> <cmd> [args...] Управление другим элементом по id (анимации/контент/trigger). См. следующий раздел.

element: — управление элементами по id (самое важное)

Команда вида: 

element: <elementId> <command> <args...>

Команды element:

Command Пример Что делает
anim element: logo anim pause Управление анимацией элемента (pause/resume/stop/stop_forever/start/restart).
text element: title text &aНовый текст Меняет базовый текст у TEXT (применяет color()).
image element: logo image logo_2 Меняет базовую картинку у IMAGE (item model id).
material element: cube material GOLD_BLOCK Меняет material у ITEM или BLOCK.
reset element: logo reset Сбрасывает элемент к исходным значениям из menu.yml (base/original).
trigger element: btn trigger left Запускает actions другого элемента (left/right/onHover/onUnhover).

Примеры actions (готовые рецепты)

1) Кнопка останавливает анимацию у другого элемента

- id: logo
  type: IMAGE
  x: 0
  y: 1
  z: 0
  image: "logo_1"
  animation: "logo_anim"

- id: stopBtn
  type: TEXT
  x: 0
  y: 0.4
  z: 0
  text: "&cSTOP"
  left:
    - "element: logo anim stop_forever"

- id: startBtn
  type: TEXT
  x: 0
  y: 0.2
  z: 0
  text: "&aSTART"
  left:
    - "element: logo anim restart"

2) Наведение на кнопку ставит лого на паузу

- id: play
  type: TEXT
  x: 0
  y: 0.2
  z: 0
  text: "&aPlay"
  onHover:
    - "element: logo anim pause"
  onUnhover:
    - "element: logo anim resume"

3) “Событие” через trigger

- id: button
  type: TEXT
  text: "&eDo it"
  left:
    - "element: target trigger left"

- id: target
  type: TEXT
  text: "&7Target"
  left:
    - "message: &aTarget LEFT triggered!"

Анимации — обзор

Анимации лежат в plugins/Menuholograms/animations. Любой файл .yml считается анимацией. Имя анимации = имя файла без расширения.

Формат файла анимации (каждый параметр)

Параметр Тип Как работает
mode single / repeat / lastframe
  • single — проигрывает все кадры один раз и возвращает базовый визуал.
  • repeat — зацикливает кадры.
  • lastframe — проигрывает и останавливается на последнем кадре.
type TEXT / BLOCK / IMAGE К чему применяется анимация: TEXT меняет текст, BLOCK меняет material у BlockDisplay, IMAGE меняет imageId у ItemDisplay(PAPER).
content list Список кадров: значение + длительность в тиках.

Примеры

# animations/welcome_text.yml
mode: repeat
type: TEXT
content:
  - "&aПривет": 10
  - "&eДобро пожаловать": 10
  - "&bНа сервер": 10
# animations/logo_anim.yml
mode: repeat
type: IMAGE
content:
  - "logo_1": 5
  - "logo_2": 5
  - "logo_3": 5
# animations/block_cycle.yml
mode: repeat
type: BLOCK
content:
  - "STONE": 10
  - "GOLD_BLOCK": 10
  - "DIAMOND_BLOCK": 10
Важно: type должен совпадать с типом элемента (TEXT↔TEXT, IMAGE↔IMAGE, BLOCK↔BLOCK), иначе анимация не применится.

Привязка анимации к элементу

- id: logo
  type: IMAGE
  x: 0
  y: 1
  z: 0
  image: "logo_1"
  animation: "logo_anim"
  animation_autostart: true
Если animation_autostart не указан: считается true.

Управление анимацией

Через hover (на том же элементе)

hover:
  animation_action: pause   # pause | stop | stop_forever | start | restart
animation_action Что происходит Что на unhover
pause Анимация “замораживается” на текущем кадре. Продолжается с того же кадра.
stop Анимация останавливается и визуал возвращается к базовому. Запускается заново (с начала), если не остановлена навсегда.
stop_forever Анимация полностью отключается. Не восстановится сама (нужно element: ... anim restart).
start Запускает анимацию если она не идёт. Ничего особенного.
restart Перезапускает анимацию с первого кадра. Ничего особенного.

Через actions других элементов (element: id anim ...)

left:
  - "element: logo anim pause"
  - "element: logo anim resume"
  - "element: logo anim stop"
  - "element: logo anim stop_forever"
  - "element: logo anim start"
  - "element: logo anim restart"

Hover и rayTraceEntities — как это реально работает

Hover считается так:

  • Берётся позиция глаз игрока (eye) и направление взгляда (dir).
  • Вызывается world.rayTraceEntities(eye, dir, maxDistance, raySize, filter).
  • В filter пропускаются только те сущности, которые зарегистрированы как кликабельные (Interaction-хитбоксы).
  • Если луч попал в хитбокс — элемент считается наведённым.
Почему это важно: hover не зависит от “визуала” (Display). Если hitbox маленький или смещён — hover будет “мимо”. 
// пример параметров из логики плагина
maxDistance = 6.0
raySize     = 0.15
Если хочешь, чтобы hover работал “с дальнего расстояния”:
увеличь maxDistance в коде или делай меню ближе к игроку, а также увеличивай размеры хитбоксов.

Поворот при pitch 90/-90 — почему ломалось и как исправлено

Классическая проблема: если брать направление через player.getLocation().getDirection(), то при pitch близком к ±90 вектор становится почти вертикальным, а после обнуления Y он превращается в (0,0,0), и “направление вперёд” ломается.

Решение: направление для anchor считается только по yaw, игнорируя pitch. Тогда при любом pitch меню будет стабильно спавниться “вперёд по yaw”.

Видимость меню — почему другие игроки не видят чужие меню

Меню “персональное”: каждый Display/Interaction скрывается от всех игроков, кроме владельца.

  • При спавне сущностей плагин делает hideEntity для всех кроме viewer.
  • При заходе игрока в игру применяется “донастройка видимости”, чтобы новые игроки тоже не увидели чужие сущности.
Итог: каждое меню видно только своему игроку.

Troubleshooting (частые проблемы)

1) Hover не срабатывает

  • Проверь размеры hitbox.width/height (или добавь extra_hitboxes).
  • Убедись, что хитбокс не смещён неверно (yOffset).
  • Учти ограничение дальности луча (обычно 6 блоков).

2) Кликаю по визуалу, но не работает

  • Клики считаются по зарегистрированным сущностям (обычно Interaction + иногда визуал).
  • Если “мимо” — опять же hitbox.

3) IMAGE не отображается

  • Проверь, что ресурс-пак реально загружен игроку.
  • Проверь, что item model существует для menuholograms:<imageId>.
  • imageId приводится к lower-case.

4) Анимация не проигрывается

  • Проверь совпадение type анимации и типа элемента.
  • Проверь animation_autostart (если false — запускай через actions).
Если что-то “вдруг исчезает”: меню удаляет все сущности при close(). Если у тебя close_time маленький — оно может закрываться само.
